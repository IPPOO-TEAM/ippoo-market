1. Structure mobile des cartes produits

Disposition verticale, claire et compacte :

+-----------------------------------+
| IMAGE PRODUIT (carré, plein écran)|
| NOM DU PRODUIT                     |
| Prix unitaire                       |
| Prix lot (MOQ)                      |
| Stock disponible                     |
| Options livraison (icônes)          |
| [Bouton Commander / Ajouter au panier] |
+-----------------------------------+

Détails :

Image principale : occupe la largeur de l’écran, aspect ratio 1:1 ou 4:3.

Nom du produit : en gras, taille 16-18 px pour lisibilité mobile.

Prix et lot : prix unitaire en haut, prix lot juste en dessous avec un badge couleur (ex : vert ou bleu).

Stock / MOQ : petite ligne discrète sous le prix, par exemple “50 unités disponibles - Commande min 20”.

Options livraison : icônes simples (camion, éclair) alignées horizontalement, très visibles.

Bouton CTA : large, couleur contrastante (ex : bleu ou vert vif), texte clair “Commander en lot”.

2. Design mobile recommandé

A. Taille et lisibilité :

Largeur carte : 90-95 % de l’écran, marge 5 % de chaque côté.

Hauteur variable selon la description, mais compacte pour éviter le scrolling excessif.

Texte hiérarchisé : titre > prix > stock/MOQ > livraison > bouton.

B. Palette et typographie :

Titres : bleu foncé ou noir, taille 16-18 px.

Prix lot : vert ou orange pour attirer l’attention.

Informations secondaires (stock, MOQ) : gris foncé.

Bouton CTA : couleur vive et pleine largeur, coins arrondis pour mobile.

C. Interaction mobile :

Swipe ou scroll vertical : chaque carte occupe la largeur, affichage en liste.

Touch targets : boutons suffisamment grands (min. 44 px de hauteur).

Survol / tooltip remplacé par “tap” : pour plus de détails ou description courte.
